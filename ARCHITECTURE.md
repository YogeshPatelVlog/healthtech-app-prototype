
Architecture Pattern:
• MVVM with Feature-based Modular Architecture

Layers:
1. Presentation Layer
   - Screens (Login, Dashboard, Patient, AI Result)
2. ViewModel Layer
   - State management
   - Business logic
3. Data Layer
   - Local Storage (SQLite/Hive)
   - Remote API (Future scope)

Why this architecture?
• Easy scalability
• Separation of concerns
• Testability
• Plug-and-play backend & AI
